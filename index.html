VR 게임

Python: Space Escape - 학습형 VR 게임 프로젝트

개요

개발 기간 : 2025.03 ~ 2025.06

플랫폼 : Meta Quest 2

기술 스택 : Unity 2022, XR Toolkit, C#

프로젝트 유형 : 팀 프로젝트 (3인)

프로젝트 설명 :

외계인과 함께 Python 코딩으로 우주선을 운행해 탈출하는 학습형 VR 게임

VR 환경 내에서 직접 코드를 작성하고 물체와 상호작용하며 자연스러운 학습이 가능하도록 함

주요 기능

키보드와 모니터, 전송버튼

  • 키보드는 컨트롤러 또는 레이저 포인터를 통해 입력
  • 입력 문자는 모니터에 실시간으로 출력됨
  • 전송 버튼 클릭 시 자체 Python 파서를 통해 정답 여부 자동 판단
  • 정답 시 성공 효과, 오답 시 재입력 가능

가상 키보드 입력 및 모니터 출력

정답 입력 시 나타나는 시청각적 피드백

Python 파서 정답 처리 로직

외계인

  • JSON 기반 대사 스크립트를 코루틴을 통해 자동 진행 시간 설정
  • 상황 별 애니메이션 부여 및 기본 애니메이션 반복
  • 외계인 시야는 플레이어(HMD)를 기준으로 회전
  • 플레이어와 일정 거리 이상 떨어질 경우 따라옴
  • 충돌 시 소리와 진동 피드백이 발생하여 몰입감 증폭

대사 시스템 및 애니메이션

플레이어 추적 및 시선 처리

충돌 감지 및 햅틱

게임 단계 처리

  • Enum을 기반으로 둔 GameStageManager로 주요 스크립트들 중앙 제어
  • 각 단계는 외계인의 대사 → 사용자 입력 → 정답 판별 → 이벤트 → 다음 단계 순서로 진행
  • 게임 종료 시 문이 열리며, StartMenu로 회귀
  • 코루틴을 통해 이벤트를 비동기적으로 제어하여 부드러운 게임 플레이를 제공

마지막 단계 완료 후 문 열림 이벤트

StartMenu로 회귀

게임 단계 flowchart

세부 기능

  • 오브젝트 상호작용 : 버튼 클릭 시 이벤트 발생, 큐브 잡고 던지기 등의 상호작용
  • 힌트 콘솔 : 단계 별 힌트 제공, 힌트 콘솔과 충돌 시 힌트 출력
  • 씬 전환 : StartMenu → GameMenu → StartMenu 로의 씬 전환 시의 부자연스러움 해결
  • 사운드/햅틱 : 코루틴을 통해 각 사운드와 햅틱 별로 시간과 순서를 설정
  • UI 시스템 : VR 환경에 최적화된 직관적인 사용자 인터페이스

심박수 관리 어플리케이션

HealthCare - CJ대한통운 미래기술 챌린지 공모전

개요

개발 기간 : 2024.07 ~ 2024.08

플랫폼 : Android Studio + Wear OS

기술 스택 : Kotlin, Firebase, Android SDK

프로젝트 유형 : 팀 프로젝트(2인)

프로젝트 설명 :

물류 현장 근로자의 건강 관리를 위한 심박수 모니터링 애플리케이션

실시간 심박수 데이터를 수집하여 근로자의 건강 상태 모니터링 및 관리자의 신고 기능

기능

일반회원 모드

  • 회원가입 및 로그인, 생체 인식 로그인 시스템
  • 아이디/비밀번호 찾기 기능
  • 30일 간 최대, 평균, 최소 심박수 그래프 확인
  • 건강 이상 신고 이력 조회
  • 개인 건강 데이터 관리

관리자 모드

  • 전체 회원 심박수 데이터 모니터링 시스템
  • 마스킹된 개인정보로 신고 이력 확인 시스템
  • 위험 상황 알림 시스템
  • 회원별 심박수 임계치 설정 기능
  • 관리자 설정/해제 기능

Firebase 연동

  • 워치 앱에서 받아온 실시간 데이터베이스 동기화
  • 날짜 별 최대, 평균, 최소 심박수 저장
  • 회원 개인정보, 관리자 개인정보 관리
  • 사용자 인증 및 권한 관리

트러블슈팅

Firebase 실시간 데이터 동기화 문제 +
문제

회원의 심박수 데이터가 실시간으로 동기화되지 않는 문제 발생

시도

리스너 재설정, 네트워크 상태 확인, 데이터베이스 규칙 검토

원인

Firebase Realtime Database에서 addValueEventListener() 이상 작동

해결

ValueEventListener()를 ChildEventListener()로 변경 후 정상 작동 확인

Firebase 구조 설계 오류 +
문제

심박수 데이터가 다른 사용자 데이터와 혼합되어 표시됨

시도

Firebase Realtime Database 확인 및 구조화된 경로로 변경

원인

Firebase Realtime Database에 사용자 구분 없이 공통 경로로 데이터를 저장함

해결

Firebase 경로를 users/{userId}/heartRate 형태로 분리하여 사용자별로 데이터가 저장되도록 수정
