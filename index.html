VR 게임

Python: Space Escape - 학습형 VR 게임 프로젝트

개요

개발 기간 : 2025.03 ~ 2025.06

플랫폼 : Meta Quest 2

기술 스택 : Unity 2022, XR Toolkit, C#

프로젝트 유형 : 팀 프로젝트 (3인)

프로젝트 설명 :

외계인과 함께 Python 코딩으로 우주선을 운행해 탈출하는 학습형 VR 게임

VR 환경 내에서 직접 코드를 작성하고 물체와 상호작용하며 자연스러운 학습이 가능하도록 함

주요 기능

키보드와 모니터, 전송버튼

  • 키보드는 컨트롤러 또는 레이저 포인터를 통해 입력
  • 입력 문자는 모니터에 실시간으로 출력됨
  • 전송 버튼 클릭 시 자체 Python 파서를 통해 정답 여부 자동 판단
  • 정답 시 성공 효과, 오답 시 재입력 가능

가상 키보드 입력 및 모니터 출력

정답 입력 시 나타나는 시청각적 피드백

Python 파서 정답 처리 로직

외계인

  • JSON 기반 대사 스크립트를 코루틴을 통해 자동 진행 시간 설정
  • 상황 별 애니메이션 부여 및 기본 애니메이션 반복
  • 외계인 시야는 플레이어(HMD)를 기준으로 회전
  • 플레이어와 일정 거리 이상 떨어질 경우 따라옴
  • 충돌 시 소리와 진동 피드백이 발생하여 몰입감 증폭

대사 시스템 및 애니메이션

플레이어 추적 및 시선 처리

충돌 감지 및 햅틱

게임 단계 처리

  • Enum을 기반한 GameStageManager로 주요 스크립트들 중앙 제어
  • 각 단계는 외계인의 대사 → 사용자 입력 → 정답 판별 → 이벤트 → 다음 단계 순서로 진행
  • 게임 종료 시 문이 열리며, StartMenu로 회귀
  • 코루틴을 통해 이벤트를 비동기적으로 제어하여 부드러운 게임 플레이를 제공

마지막 단계 완료 후 문 열림 이벤트

StartMenu로 회귀

게임 단계 flowchart

세부 기능

  • 오브젝트 상호작용 : 버튼 클릭 시 이벤트 발생, 큐브 잡고 던지기 등의 상호작용
  • 힌트 콘솔 : 단계 별 힌트 제공, 힌트 콘솔과 충돌 시 힌트 출력
  • 씬 전환 : StartMenu → GameMenu → StartMenu 로의 씬 전환 시의 부자연스러움 해결
  • 사운드/진동 : 코루틴을 통해 각 사운드와 햅틱 별로 시간과 위치를 설정
  • UI 시스템 : VR 환경에 최적화된 직관적인 사용자 인터페이스

Oculus Link 연결 문제 +
문제:

Oculus Quest 2와 PC 간 연결이 불안정하여 VR 개발 테스트가 어려웠음

원인:

Oculus 설정 디렉토리의 캐시 파일 충돌

시도한 방법:

USB 케이블 교체, 드라이버 재설치, Oculus 소프트웨어 재설치

최종 해결:

설정 디렉토리 삭제 후 재설정으로 연결 안정화

교훈:

하드웨어 연결 문제는 소프트웨어 캐시 초기화로 해결될 수 있음

외계인 충돌 버그 +
문제:

외계인 NPC와 플레이어 간 충돌 감지가 정상적으로 작동하지 않음

원인:

Rigidbody 컴포넌트의 잘못된 설정 및 Collider 크기 문제

시도한 방법:

Collider 크기 조정, Layer 설정 변경, Physics Material 적용

최종 해결:

Rigidbody의 Is Kinematic 설정 조정 및 Trigger 방식으로 변경

교훈:

Unity 물리 시스템의 기본 설정을 정확히 이해하는 것이 중요

키보드 입력 오류 +
문제:

VR 환경에서 가상 키보드 입력이 간헐적으로 인식되지 않음

원인:

TextMeshPro 컴포넌트의 비동기 처리 문제 및 입력 이벤트 충돌

시도한 방법:

입력 딜레이 추가, 이벤트 리스너 재설정, UI 업데이트 순서 조정

최종 해결:

코루틴을 활용한 비동기 처리 개선 및 입력 버퍼 시스템 구현

교훈:

VR 환경에서는 일반적인 UI 처리와 다른 접근이 필요함

XR 초기화 버그 +
문제:

XR 시스템 초기화 실패로 인한 앱 크래시 발생

원인:

XR Toolkit 버전 호환성 문제 및 초기화 순서 오류

시도한 방법:

XR Toolkit 버전 다운그레이드, 초기화 코드 순서 변경

최종 해결:

안정적인 XR Toolkit 버전 사용 및 예외 처리 강화

교훈:

VR 개발 시 안정적인 버전의 라이브러리 사용이 중요

Unity 버전 호환성 문제 +
문제:

팀원 간 Unity 버전 차이로 인한 프로젝트 호환성 문제

원인:

Unity 버전 간 API 변경사항 및 패키지 의존성 차이

시도한 방법:

버전 통일, 패키지 매니저 설정 동기화, 프로젝트 설정 공유

최종 해결:

Unity 2022 LTS로 통일하여 안정성 확보

교훈:

팀 프로젝트에서는 개발 환경 통일이 필수적
